Getting Started

Overview

This documentation package gives developers production-oriented patterns for building and extending the F35 Crash interface with consistent visuals and behavior.

Usage Model

  • Use design tokens first (colors, space, typography).
  • Build UI from documented component classes.
  • Keep state classes explicit for predictability.

Components

Game Components

F35-specific UI pieces and state naming convention.

Bet Panel States

  • state-bet: ready to place wager
  • state-accepted: bet queued/accepted
  • state-cashout: live payout action
  • state-drop: pod action available
  • state-win: cashout success amount
  • state-saved: pod saved amount

Power Card States

  • .active-addon: purchased and active
  • .disabled: not available for this phase
  • .docked: moved inside accepted card
  • .hidden: unavailable state
<div class="action-stack pod-mode pod-before-save">
  <button class="bet-btn state-drop">
    <span class="btn-line-one">DROP POD</span>
    <span class="btn-line-two">$34.67</span>
  </button>
  <button class="bet-btn state-cashout" disabled>
    <span class="btn-line-one">CASH OUT (NEXT)</span>
  </button>
</div>
