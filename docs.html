F35 Crash Reference

A real-time crash game built with vanilla JavaScript, Web Audio API, and Canvas. This reference covers the full architecture, game loop, betting system, audio synthesis engine, and visual renderer.

Vanilla JS No build step Web Audio API Canvas 2D Provably Fair

Overview

F35 Crash is a multiplayer-style crash gambling game where players place bets before each round. A multiplier climbs from 1.00× and crashes at a provably-fair random point. Players must cash out before the crash to collect winnings.

The entire client runs as a single-page application across four files with zero external runtime dependencies. All audio is synthesized at runtime via the Web Audio API — no audio files are bundled.

Key Concepts

  • Round — one complete taxi → flight → crash cycle
  • Multiplier — grows exponentially during the flight phase
  • Crash point — the multiplier at which a round ends
  • Cash out — locking in winnings at the current multiplier
  • Pod — a booster that auto-cashes out a portion of your stake
  • Hypersonic — a booster that guarantees a high crash point

Game Phases

Every round progresses through three phases stored in gameState.phase.

taxi
5 s · bets open
→
flying
7–15 s · multiplier climbs
→
ended
1.5 s · crash shown
→
taxi
next round
PhaseDurationWhat Happens
"taxi" TAXI_DURATION_SEC (5 s) Jet taxis on runway. Bets are accepted. Countdown ring animates. Opponents place bets.
"flying" 7–15 s (random) Multiplier climbs at 8%/s. Players can cash out. Jet ascends. Pod and Hypersonic boosters are active.
"ended" ENDED_DURATION_SEC (1.5 s) Crash multiplier is revealed. Uncashed bets are lost. Jet shows crash/fly-away animation.

Phase Transitions

// Transitions are driven by phaseElapsed inside gameLoop()
if (gameState.phase === 'taxi' && gameState.phaseElapsed >= TAXI_DURATION_SEC) {
  startRound();        // → 'flying'
}
if (gameState.phase === 'flying' && gameState.phaseElapsed >= gameState.crashAtSec) {
  onCrash();           // → 'ended'
}
if (gameState.phase === 'ended' && gameState.phaseElapsed >= ENDED_DURATION_SEC) {
  startTaxi();         // → 'taxi' (new round)
}

Game Loop

The game runs on a requestAnimationFrame loop. Each tick advances game time, updates the multiplier, triggers phase transitions, drives opponent cashouts, and re-renders the canvas. 

function gameLoop(timestamp) {
  const deltaSec = (timestamp - lastTime) / 1000;

  // 1. Advance phase timer
  gameState.phaseElapsed += deltaSec;

  // 2. Update multiplier (flying phase only)
  if (gameState.phase === 'flying') {
    gameState.multiplier = calcMultiplier(gameState.phaseElapsed);
    runOpponentCashoutTick();
  }

  // 3. Check for phase transitions
  checkPhaseTransition();

  // 4. Update DOM and canvas
  updateGameUi(timestamp);
  renderGameplayVisual(deltaSec);

  gameRaf = requestAnimationFrame(gameLoop);
}

Multiplier Formula

The multiplier grows exponentially based on elapsed seconds in the flying phase: 

// MULTIPLIER_GROWTH_RATE = 0.08 (8% per second)
multiplier = Math.pow(1 + MULTIPLIER_GROWTH_RATE, phaseElapsed * 10)

Betting System

placeBet () → void taxi phase only

Places the player's bet using gameState.betAmount. Deducts from balance, sets betPlaced = true, and adds the player row to the Live Bets panel. Rejected if phase is not "taxi" or balance is insufficient.

cashOutBet (betId: string, source: string) → void flying phase only

Cashes out a bet at the current multiplier. Calculates payout as stake × multiplier, updates balance, marks the bet row as cashed out, and triggers the cashout SFX. source is used for logging ('player' | 'pod' | 'auto').

handleBetButtonAction () → void

Unified handler for the main bet button. Behavior changes based on phase and state:

  • taxi, no bet → calls placeBet()
  • taxi, bet placed → cancels the bet and refunds balance
  • flying, active bet → calls cashOutBet()
updateBetAmount (next: number) → void

Sets gameState.betAmount to next and refreshes the bet input display. The ½ and 2× buttons call this with betAmount / 2 and betAmount * 2 respectively.

getCurrentPotentialWin () → number

Returns the hypothetical payout if the player were to cash out right now: stake × gameState.multiplier. Used to populate the "potential win" display in the control panel.

Power-ups

Pod

Costs $10. When active, a pod detaches from the jet mid-flight and automatically cashes out 50% of the stake at a random safe multiplier (roughly 1.3–2.5×). The remaining 50% continues riding the main multiplier.

buyPod () → void $10 cost

Deducts $10, sets gameState.podBought = true, activates the pod UI, and schedules pod cashout. Only available when a bet is placed and the player has sufficient balance.

Hypersonic

Costs $40. Guarantees the round's crash point will be one of the preset high multipliers: [5, 10, 20, 25, 50, 100]. The jet enters a special "hypersonic" exit animation when active.

buyHypersonic () → void $40 cost

Deducts $40, picks a random value from HYPERSONIC_MULTIPLIERS, overrides gameState.crashAtSec, and sets hypersonicActive = true. Only available during the taxi phase with an active bet.

Power-ups are one-time-per-round purchases. They reset at the start of each new taxi phase.

Provably Fair

Each round's crash point is derived from a server hash, client seed, and nonce using standard cryptographic primitives — allowing anyone to verify the result independently.

Algorithm

function calculateCrashPoint(serverHash, clientSeed, nonce) {
  // 1. Combine inputs into a deterministic seed
  const combined = serverHash + clientSeed + nonce;

  // 2. Hash the combined seed
  const hash = hashString(combined);

  // 3. Normalize to [0, 1]
  const normalized = hash / MAX_HASH_VALUE;

  // 4. Map to crash multiplier (minimum 1.00×)
  return Math.max(1.0, 1 / (1 - normalized));
}
calculateCrashPoint (serverHash: string, clientSeed: string, nonce: number) → number

Returns the provably-fair crash multiplier for a given set of seeds. The player can supply their own clientSeed via the Provably Fair Settings modal to influence future rounds.

fnvHash (input: string) → number

32-bit FNV-1a hash. Used for deterministic crash history generation and client-side seed hashing.

Round History Verification

Each history row exposes the server seed hash and nonce. Players can verify any past round by entering these values in the Provably Fair modal, which re-runs calculateCrashPoint() and compares to the recorded result.

All audio is synthesized in real time using the Web Audio API. There are no audio files — every sound is generated from oscillators, noise buffers, and filters.

Initial opponent roster. Each entry defines a username and stake amount used to populate betsState at the start of each round.

Pool of chat messages used by the reaction simulation engine.
